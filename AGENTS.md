# Agent Bootstrap

Before making changes, read project context in this order:

1. `Docs/INDEX.md`, followed by every existing Foundation Document in the order it defines.
2. `DEVELOPMENT_CONTRACT.md`.
3. `Docs/CURRENT_STATE.md`.
4. `Docs/CURRENT_TASK.md`.

Then read only the implementation or execution plans relevant to the current task and inspect the current code, tests, Git state, and applicable local instructions.

Follow the current task boundary, make the smallest coherent change, validate its success criteria, update the two current-context documents when the handoff changes, and stop at the batch boundary. Use the Development Contract for authority, completion, and escalation rules. Foundation Documents remain authoritative for product decisions.
